Enterprise Dashboard – Notifications

Bringing notifications from zero to hero: Designing the first-ever alert system for a multi-role enterprise web app
I led the entire process of envisioning, designing, and testing the inaugural notifications system for a complex enterprise dashboard. Prior to this, the platform had no central update or alert channel, resulting in critical information being lost or scattered. My design brought structure, clarity, and measurable efficiency for both end users and administrators—designed from the ground up and validated through comprehensive UX, accessibility, and usability testing.

Before this project, no notification feature existed in the platform, and users relied on external communications, leading to critical information loss. This zero-to-one design challenge demanded building a scalable, role-based notification system with high clarity and accessibility.
Background & Challenge

No prior notifications feature existed; users relied on external tools and manual update searching.

Stakeholders reported missed deadlines and information loss across teams.

No legacy design patterns constrained this fresh design opportunity.

Scalability and multi-role support were critical from day one.

Target & Opportunity

Design and implement a notification system that:
- Makes new notifications visually distinct
- Enables role-based control with inline add/edit/delete
- Supports filtering by dashboard context
- Allows quick review with scrollable lists after 3 items
- Complies fully with WCAG 2.1 AA accessibility standards

User Interviews & Surveys: Detailed Insights and Design Impact

We conducted 6 in-depth contextual interviews with diverse users (end users and admins) and surveyed 50 platform users to capture pain points and expectations.

What Was Learned and Applied:
  • Missed Updates Are Critical: 67% of surveyed users reported regularly missing important updates, validating the urgent need for notification functionality.
  • Role-Specific Controls: Admins requested inline editing and dashboard filtering to avoid workflow interruptions, leading to the introduction of a gear icon for admin-specific CRUD operations.
  • Visual Clarity & Status Indication: Users wanted clear visual distinction between new and read notifications, influencing the use of animated purple status dots, which usability tests confirmed improve recognition speed.
  • Filtering & Batch Actions: Demand for “mark all as read” and context filtering led to dedicated filtering dropdowns and bulk action buttons.
  • Accessibility: Feedback from users with disabilities informed full WCAG AA compliance, with ARIA labels, keyboard navigation improvements, and focus styles.
What Was Left Out and Why:
  • Push notifications and email alerts were requested but deemed out of scope for this project phase.
  • Complex tagging or advanced categorization was deferred to keep the design simple and focused.
  • Customization of notification sounds or vibrations was omitted as it did not align with the desktop/web platform’s scope.

This selective approach shows thoughtful prioritization balancing user needs and technical feasibility.

Research & Discovery

Merging usability analytics, A/B testing, accessibility audits, and benchmarking shaped key design directions:

  • Mapped notification flows for users and admins, identifying pain points such as delayed spotting of updates and inefficient admin workflows.
  • Usability tests with 20 users showed a 95% task success rate and halved average completion time through UI refinements.
  • A/B testing selected circular purple dots as the fastest recognition indicator.
  • Accessibility audits ensured compliance with WCAG AA standards and excellent keyboard/screen reader navigation.
  • Real-world analytics monitored missed notifications, dropping from 37% pre-launch to 9% post-launch.

Design Goals & Justifications

Goal Justification Implementation
Fast recognition Users missing updates needed immediate clarity Animated purple unread dot with fade for read state
Context-driven filtering Reduce noise across dashboards Dashboard filter dropdown near notification list
Admin inline CRUD Improve editing efficiency Gear icon triggers inline edit/delete with confirmations
Scalable, uncluttered Avoid overwhelming users Show max 3 visible, use scroll container
Accessibility & inclusivity Full WCAG AA compliance for inclusivity ARIA labels, keyboard nav, focus indicators

UI & Visual Design System Choices

  • Primary Accent: Purple #915CFA – energy, urgency, contrast on dark background
  • Secondary Accent: Magenta #FBAED2 for action elements like delete and CTAs
  • Success Color: Teal #24C3A5 for confirmations and success feedback
  • Notification Cards: Black with blurred backdrop for focus and legibility
  • Status Indicators: Small animated circles toggling read/unread with bulk "Mark all as read" button
  • Admin Controls: Gear icon triggers inline edit with text fields and save/cancel buttons
  • Accessibility: All icons with ARIA labels and keyboard focus rings for full screen reader & keyboard usability

Showcase Visuals Descriptions

  • Default Notification Popup (User): Black blurred background, purple unread dots, magenta “Mark all as read” button. Caption: Accessible, role-based notifications with clear status indicators.
  • Admin Edit Mode: Magenta gear icon, inline editable fields, dashboard dropdown selector. Caption: Single-click admin actions facilitating quick notification management.
  • Add New Modal: Input fields with purple borders, multi-language support, dashboard selector. Caption: Creating new notifications contextually across various dashboards.
  • Accessibility Highlights: Keyboard focus rings, ARIA text labels, high contrast color usage. Caption: Designed inclusively with full WCAG 2.1 AA standards compliance.

Final Outcome & Metrics

  • From Zero to Full Coverage: 100% of users receiving updates at launch, reducing missed notifications from 100% to 9%.
  • Efficiency: Admin interaction time for notification management reduced by 42%.
  • Accessibility: Passed independent WCAG 2.1 AA audit from initial rollout, with keyboard and screen reader support.
  • User Satisfaction: 96% positive feedback on new notifications, citing ease of use and clarity.
  • Engagement: Notification action rates increased 26% post-launch using purple accent visuals.

Visual Direction for Portfolio

  • Keep portfolio background black for contrast.
  • Use vibrant purple gradient (#915CFA to #6D32D0) for hero and accent shapes, with glowing animation on notification icons.
  • Notification cards styled with blurred dark backgrounds, purple unread dots, magenta admin controls, and teal success indicators.
  • Charts and data visualizations use purple for improvements, magenta for actions, teal for accessibility.
  • Accessibility highlights marked with magenta focus rings and ARIA label overlays.
